Details of "Small Shooter"

Platform: Windows

Game objectives and conditions to be aware of:

In the game, you play as the circle that moves around.
You can shoot the green enemy blobs, and recieve points for everything you kill
You have 3 lives, and everytime you get hit by an enemy your health goes down by 1
When your health reaches 0 you die, and restart the game

Controls

WASD/UpDownLeftRight - move
Mouse movement - aiming
Left click - shoot


Code Details

SOURCE CODE AND HEIRARCHIES WILL BE WRITTEN AND SHOWN IN LABLED SNIPPETS/IMAGES WITH COMMENTS DESCRIBING WHAT THEY DO

Room One

The game begins in the title room, with a background with 1 object,
the background is a tileset, which is a fancy word for a sprite that was
up into several small sprites that can interactively be drawn within the
on the background.
The object is a simple title control object, which initialized through a simple function that
increases the opacity until a cap of 1 (Fig A.), then the title object checks every frame for the
endter key to be released. Upon release the enter key will push the game into the next room


Room Two

The second room begins with a player object, an object to control score, and an object to control enemy spawn as well as bullets that the player fires that damage enemy health
All rooms have the tileset based background from the previous room and I will omit discussing them henceforth


The player object // obj_player

This object is the player, and moves and shoots on command. (For in depth code, see Fig B.)

The enemy control object // obj_control

This object is intended to create enemy objects at random places in the game on a timer. (For in depth code, see Fig B.)

The score object // obj_score

This object draws the score on every frame. (Fig B.)


For all heirarchies and visuals see Fig C.
